# /docs/08_prioritization

This repository contains the **MVP definition and prioritization documents** for a media catalog application inspired by platforms such as Letterboxd or Serializd.

The goal of this project is to define a **realistic, scoped MVP** for an application that allows users to track, rate, and review books, video games, movies, and TV series.

---

## Documents Overview

### 1. `mvp_definition.md`

This document provides a **precise definition of the MVP scope**.  
It describes **what is included in the MVP and what is explicitly excluded** in order to keep the first version focused and achievable.

It contains:

- **Main user flows**
- **Essential screens**
- **Minimal feature set**
- **Out-of-scope features**

This document answers the question:

> _What is the smallest product that still delivers value to users?_

---

### 2. `prioritization_matrix.md`

This document explains **how and why features were prioritized** for the MVP.

It uses a prioritization framework (such as **MoSCoW** or **RICE**) to:

- justify feature selection
- distinguish **must-have** features from **nice-to-have** ones
- ensure the MVP remains feasible within limited time and resources

This document answers the question:

> _Why are these features included in the MVP and not others?_
