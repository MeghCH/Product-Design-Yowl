## Test Protocol – FEEDBACK

## Objective

The goal of this test is to evaluate the usability and navigation of the prototype on both desktop and mobile versions.  
Testers are asked to follow a predefined user journey in order to identify any difficulties, confusion, or usability issues.

## Context

Testers should use the prototype as if they were real users discovering the application for the first time.  
They should speak out loud when possible and explain what they think each button or page is for.

---

## Test Tasks – Desktop Version

1. Start on the home page while not logged in.
2. Log yourself.
3. Once on your profile page, go to the books list of your profil.
4. Get more information about a book.
5. Write a review of this book..
6. In the reviews section, access another user’s profile.
7. Go back to the home page.
8. Go back to your profile.
9. Log out.

## Observation Criteria

During the test, observe and note:

- If the tester hesitates or gets lost
- If buttons are easy to understand and find
- If navigation feels logical
- Any comments or frustrations expressed by the tester
- Any actions that do not match the intended user journey
