# /docs/07_feedback

This folder contains the documentation related to user testing of the prototype.

## Contents

### test_protocol.md

This file describes the testing scenario given to the testers.  
It explains:

- The goal of the test
- The context in which testers used the prototype
- The tasks they were asked to complete on desktop and mobile versions

This document serves as a guide to understand what actions testers were instructed to perform.

### feedback_summary.md

This file contains the consolidated feedback from 3 to 5 testers (classmates).  
It includes:

- Notes collected during or after the tests
- Recurring problems and usability issues
- Points of confusion
- Opportunities for improvement

The feedback is grouped by themes to highlight patterns rather than individual opinions.

## Purpose

The goal of this folder is to:

- Show how the prototype was tested
- Demonstrate a user-centered design approach
- Identify usability problems
- Provide a base for future improvements and prioritization
